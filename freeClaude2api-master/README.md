# freeClaude2api

Локальный прокси-балансировщик, который даёт настоящему **Claude Code CLI** работать через пул ключей [FreeModel](https://freemodel.dev) с автоматической ротацией, кулдаунами и веб-дашбордом.

В отличие от подходов, которые пытаются *подделать* Claude Code вручную, здесь работает **сам официальный `claude` CLI** — прокси лишь подменяет API-ключ на свободный из пула и форвардит запрос на `cc.freemodel.dev`. Поэтому запросы выглядят для апстрима полностью легитимными.

```
claude CLI  ──►  freeClaude2api :8742  ──►  cc.freemodel.dev
(настоящий)      подмена x-api-key            (видит легитимный
                 + ротация по пулу             Claude Code запрос)
```

---

## Возможности

- **Прозрачный прокси** для Anthropic Messages API (`/v1/messages` и любые пути) на `cc.freemodel.dev`.
- **Пул ключей с ротацией.** Запрос уходит на первый свободный ключ; при `429` ключ уходит в кулдаун, а запрос автоматически ретраится на следующем.
- **Умные кулдауны.** Длительность блокировки парсится из заголовков (`retry-after`, `*-reset`) или из текста ошибки; если ничего нет — дефолт 5 часов.
- **Пин ключа.** Можно «прибить» конкретный ключ к сессии, передав его в `x-api-key`.
- **Запуск Claude в один клик.** Кнопка на дашборде стартует `claude` в выбранной папке с уже настроенными `ANTHROPIC_BASE_URL`/`ANTHROPIC_API_KEY`.
- **Веб-дашборд** на `/`: статус ключей, аптайм, графики нагрузки/латентности, лог запросов, добавление/удаление ключей, ручной сброс кулдауна, тест всех ключей.
- **Авто-проверка** ключей в фоне каждые 10 минут.
- **Хранилище** — локальный SQLite (`fms.db`).

> Поддерживаемые апстримом модели: `claude-opus-4-7`, `claude-sonnet-4-6`, `claude-haiku-4-5-20251001`. GPT и другие OpenAI-модели через этот эндпоинт **не** работают — для них есть отдельный проект [freeGPT2API](https://github.com/Asati-Privatka/freeGPT2API).

## Требования

- **Python 3.10+**
- **Claude Code CLI** на PATH — [инструкция по установке](https://docs.claude.com/claude-code). Без него прокси стартует, но кнопка «Launch Claude» работать не будет.
- Зависимости: `fastapi`, `uvicorn`, `httpx`, `brotli`, `zstandard`.

## Установка и запуск

### Windows (просто)
Дважды кликните **`run.bat`** — он проверит Python/pip, доустановит зависимости, убьёт прежний процесс на порту и откроет дашборд.

### Вручную (любая ОС)
```bash
pip install -r requirements.txt
python proxy.py
```

После старта:
- Дашборд / прокси: <http://localhost:8742/>

## Настройка ключей

1. Скопируйте пример и впишите свои ключи:
   ```bash
   cp keys.txt.example keys.txt
   ```
   По одному ключу на строку. Ключи из `keys.txt` импортируются в БД при первом запуске.
2. Либо добавьте ключи прямо в дашборде (поле **Add key**) — это основной способ после первого старта.

Где взять ключи: регистрация на [freemodel.dev](https://freemodel.dev) (можно через одноразовую почту [temp-mail.org](https://temp-mail.org)).

## Как пользоваться

1. Запустите прокси.
2. Добавьте один или несколько FreeModel-ключей.
3. Нажмите **Launch Claude**, укажите рабочую папку — откроется консоль с `claude`, уже настроенным на прокси.

Или вручную направьте любой Claude-клиент на прокси:
```bash
set ANTHROPIC_BASE_URL=http://localhost:8742
set ANTHROPIC_API_KEY=proxy-managed
claude -p "Привет!"
```
`ANTHROPIC_API_KEY` может быть любым (`proxy-managed`) — прокси сам подставит реальный ключ из пула. Если указать в нём конкретный ключ из пула, он будет «запинен» на эту сессию.

## API дашборда

| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/_api/status` | Сводка: ключи, лог, аналитика |
| `POST` | `/_api/keys/add` | Добавить ключ (`{"key": "..."}`) |
| `DELETE` | `/_api/keys/{index}` | Удалить ключ |
| `DELETE` | `/_api/keys/{index}/cooldown` | Сбросить кулдаун ключа |
| `POST` | `/_api/keys/test` | Проверить все ключи против апстрима |
| `POST` | `/_api/run` | Запустить `claude` в папке (`{"dir": "...", "keyIndex": N?}`) |
| `POST` | `/_api/log/clear` | Очистить лог и счётчики |

Всё остальное (`/{path}`) прозрачно проксируется на `cc.freemodel.dev`.

## Структура

```
.
├── proxy.py            # FastAPI-прокси: роутинг, ротация, кулдауны, дашборд-API
├── db.py               # SQLite-хранилище ключей и кулдаунов
├── requirements.txt
├── run.bat             # Windows-лаунчер с проверкой окружения
├── keys.txt.example    # шаблон файла ключей
├── keys.txt            # ваши ключи (в .gitignore)
├── fms.db              # БД (в .gitignore)
└── static/             # дашборд (HTML/CSS/JS)
```

## Замечания по безопасности

- Прокси слушает **только `127.0.0.1`** — наружу не выставлен.
- `keys.txt`, `fms.db`, `cooldowns.json` и логи в `.gitignore` — **не коммитьте ключи**.
- `/_api/status` отдаёт полные ключи в JSON, но только на localhost.
- Сервис обходит лимиты FreeModel ротацией ключей — учитывайте ToS провайдера.
